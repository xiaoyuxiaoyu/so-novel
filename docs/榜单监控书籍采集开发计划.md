# Phase 2 & Phase 3 开发计划

> 基于《AI选书投放工作流方案》Phase 2（M16-M25）+ Phase 3（M26-M28）

## 总体概述

- **Phase 2**：书籍自动采集 + AI预筛（新应用 `apps/book_collector/`）+ 本地下载代理（`tools/download_agent/`）
- **Phase 3**：自动评估流水线（扩展 `apps/book_selection/`）+ 推荐书单

两阶段共 13 个里程碑（M16-M28），按依赖关系分为 **服务器端模型层 → 爬虫层 → 服务/API层 → 本地代理 → AI能力 → 自动流水线** 渐进交付。

**BaseCrawler 接口**：

```python
class BaseCrawler(ABC):
    """榜单爬虫基类"""

    @abstractmethod
    async def crawl_list(self, params: CrawlParams) -> list[BookMeta]:
        """抓取书单（榜单/分类），返回标准化的书籍元数据列表"""

    @abstractmethod
    def get_available_rankings(self) -> list[RankingInfo]:
        """返回该平台支持的榜单列表"""
```

**BookMeta 数据结构**：

```python
@dataclass
class BookMeta:
    source_platform: str      # qimao
    source_id: str            # 平台书籍ID
    source_url: str           # 原始链接
    title: str
    author: str
    gender: str               # male / female
    category: str             # 都市、玄幻、言情 等
    description: str          # 简介
    word_count: int           # 总字数
    chapter_count: int        # 总章节数
    status: str               # ongoing / completed
    popularity_metrics: dict  # 阅读量、收藏、推荐票等
    tags: list[str] | None    # 平台标签
```

**QimaoCrawler 实现要点**：
- 使用 `httpx.AsyncClient` 发请求
- `beautifulsoup4 + lxml` 解析HTML
- `fake-useragent` UA轮换
- 请求间隔 3-5 秒（随机）
- 失败重试最多 3 次，指数退避
- 支持热门榜、男频/女频分类榜（各题材）、完本榜
- 每个榜单采集前 50 本

**实施步骤**：
1. 研究七猫榜单页面结构（URL、HTML结构、分页方式）
2. 实现 `BaseCrawler` 抽象类和 `BookMeta` 数据结构
3. 实现 `QimaoCrawler`（解析逻辑 + 反爬措施）
4. 编写测试（mock HTTP 响应，验证解析正确性）

**新增依赖**：

**去重过滤逻辑**（在 `upsert_from_crawl` 中）：

```
入池前检查（按顺序）：
1. source_platform + source_id 已存在 → 更新排名记录，跳过入池
2. title + author 已存在（跨平台去重） → 仅标记关联，不重复入池
3. word_count < 5万字 → 跳过
4. chapter_count < 500 → 跳过
5. 已在书库 bs_library_books 中 → 跳过
6. 通过所有检查 → 新增入池（crawl_status=meta_only）
```

**agent.py 主循环**：

```python
async def main():
    while True:
        # 1. GET /api/book-collector/agent/tasks → pending 任务
        tasks = await pull_tasks()

        for task in tasks:
            # 2. 搜索：so-novel /search/aggregated?kw=书名(作者)
            await report_searching(task.id)
            search_result = await so_novel.search(task.title, task.author)
            best_match = select_best_match(search_result, task.title, task.author)
            await report_search_result(task.id, best_match.url, best_match.source)

            # 3. 下载：so-novel /book-fetch
            txt_path = await so_novel.download(best_match.url, scope=task.scope, ...)

            # 4. 拆分
            chapters = chapter_splitter.split(txt_path)

            # 5. 校验
            chapters = content_validator.validate(chapters)

            # 6. 回传
            await report_result(task.id, chapters)

        # 7. 检查监控任务
        monitors = await pull_monitors()
        for mon in monitors:
            # 增量下载逻辑...

        await asyncio.sleep(POLL_INTERVAL)  # 默认60秒
```

**content_validator.py**：

| 校验项 | 检测方式 | quality_issues 标记 |
|--------|---------|-------------------|
| 章节连续性 | chapter_no 是否连续无跳号 | `{"type": "missing_chapter", "missing": [15, 16]}` |
| 异常章节 | word_count < 200 | `{"type": "too_short", "word_count": 56}` |
| 乱码检测 | 非CJK字符占比 > 30% | `{"type": "garbled", "non_cjk_ratio": 0.45}` |

- suspicious 章节数 > 总章节 10% → 整本标记 `partial`

## Phase 3：自动评估流水线

### 推荐书单与现有书库管理的关系

**书库（`bs_library_books`）= 要投放的书**。书库关联了投放映射、投放跟踪、投放汇总等下游模块，入库意味着"这本书要做了"（翻译、上架、投放）。

**推荐书单（`bs_recommendations`）= AI觉得不错的书**。自动评估产出的推荐，编辑确认只是认可"质量可以"，但不代表要立刻做这本书。

**两者是两个独立决策节点，不自动串联**：

```
自动流程：
  候选书池 → 每晚自动评估 → B级以上 → 推荐书单(bs_recommendations)
                                         ↓ 编辑确认(approve)
                                    标记"质量认可"（仅状态变更，不入库）
                                         ↓ 编辑后续决定要做这本书
                                    手动"加入书库" → bs_library_books

手动流程（现有，保持不变）：
  编辑手动创建评估 → 评估列表 → 手动"加入书库" → bs_library_books
```

**推荐书单确认 ≠ 入库**，入库始终是编辑的独立决策：
- 推荐书单的 `approve` 只更新 `status=approved`，表示编辑认可评估质量
- 入库仍通过现有书库管理的"加入书库"操作（`LibraryService.add_to_library`）
- 推荐书单页面可提供"加入书库"快捷按钮，但本质是调用现有入库流程，需编辑主动点击

**推荐书单的独有价值**：
1. **审核状态**（pending/approved/rejected）—— 筛选出值得关注的书
2. **来源追溯**（candidate_book_id）—— 知道书从哪个候选书/榜单来
3. **批量审核体验** —— 编辑看精简卡片（等级+分数+市场），快速过滤
4. **与书库解耦** —— 确认推荐不触发入库，避免未做的书污染书库
